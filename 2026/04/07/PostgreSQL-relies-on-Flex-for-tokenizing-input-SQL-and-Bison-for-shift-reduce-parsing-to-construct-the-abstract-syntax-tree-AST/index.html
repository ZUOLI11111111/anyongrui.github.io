PostgreSQL relies on Flex for tokenizing input SQL and Bison for shift-reduce parsing to construct the abstract syntax tree (AST).

1.Parser Overview

(1) pg_parse_query serves as the entry point for PostgreSQL parsing, and it invokes raw_parser to carry out the actual parsing process

(2) raw_parser is responsible for building the AST. It is the core parser in PostgreSQL, implemented using Flex and Bison

raw_parser performs the following steps:

  • Uses Flex for lexical analysis (tokenization)
  • Uses Bison for syntax analysis (shift/reduce parsing)
  • Constructs the Abstract Syntax Tree (AST)
  • Returns a list of RawStmt nodes

(3) pg_analyze_and_rewrite_fixedparams is the entry point for pg_analyze_fixedparams and pg_rewrite_query. It performs semantic analysis (pg_analyze_fixedparams) and query rewriting (pg_rewrite_query). It first converts the raw parse tree into a semantically meaningful Query structure by resolving table names, column references, and data types, and then applies rewrite rules such as view expansion and rule system transformations

2.Lexical Analysis & Syntax Analysis Overview

(1)Parser is defined in scan.l and gram.y

(2)gram.y is a Bison grammar file.

It is processed by Bison to generate:

  • gram.c -> the parser implementation
  • gram.h -> token definitions

The generated parser performs:

  • Shift/Reduce parsing
  • AST construction (e.g.,RawStmt,SelectStmt)

(3)scan.l is a Flex lexer specification

It is processed by Flex to generate scan.c

The lexer is responsible for converting SQL strings into tokens (e.g.,SELECT,IDENT,FROM)

(4)Key Parsing Mechanism

  • Shift: move forward in the input,push tokens onto parsing stack
  • Reduce: Popping the right-hand side (RHS) symbols from the stack. Pushing the left-hand side (LHS) non-terminal back onto the stack.A reduce operation pops symbols and states from the stack, then uses the goto table totarnsition to a new state based on the reduced no-terminal
  • The action table in Bison is encoded in arrays such as yypact,yytable and yycheck,which together determine whether to shift, reduce, or report an error based on the current state and lookahead token
